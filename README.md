## Overview

The Shemhamphorash (שם המפורש, "the explicit Name") is the 216-letter Name of God derived by applying the ancient Boustrophedon cipher to three consecutive verses of Exodus — each containing precisely 72 Hebrew consonants. When read in alternating directions and stacked in triplets, they yield the 72 three-letter roots from which the names of the 72 Angels and their opposing 72 Goetic Spirits are formed.

This tool places all 72 Names on a 360-degree zodiacal wheel divided into 5-degree arcs, embedding each gate within its corresponding decan, zodiac sign, tribe of Israel, and YHVH permutation — then links every gate to its angel's function, choir, Psalm of invocation, birthdate range, and gematria value, alongside the corresponding spirit from the Ars Goetia and the matching verse of Sūrat al-Muṭaffifīn (Quran 83).

It is designed to serve as a primary reference and contemplative tool for practitioners of the Neo-Hermetic Order of Melchizedek.

### ✡ Sepher Yetzirah

The mathematical and geometric foundations of the Hebrew alphabet as described in the *Sepher Yetzirah* (ספר יצירה, "Book of Formation").

**Part I — The 231 Gates**

The 22 Hebrew letters arranged in a circle with every letter connected to every other yields:

```
C(22, 2) = 22 × 21 ÷ 2 = 231 Gates
```

The tab includes the mandala image of all 231 connections, an explanation of the combinatorial mathematics, a complete glyph chart of the 22 letters with their gematria values and threefold classification (3 Mothers, 7 Doubles, 12 Simples), and an account of the cosmological significance of the forward and reverse gate pairs.

**Part II — The Cube of Space**

The 22 letters are also mapped onto three-dimensional geometry:

| Group | Letters | Geometric Role |
|---|---|---|
| 3 Mother letters (א מ ש) | Alef, Mem, Shin | The 3 axes (horizontal, depth, vertical) |
| 7 Double letters (ב ג ד כ פ ר ת) | Bet–Tav | The 6 faces + the holy Palace at centre |
| 12 Simple letters (ה–ק) | Heh–Qof | The 12 edges of the cube |

The image of the Cube is included alongside explanations of each letter's directional assignment, the correspondence with the zodiac and months, and the Western Hermetic synthesis (Paul Foster Case / B.O.T.A.) that maps the Major Arcana of the Tarot onto the same geometric structure.

## Getting Started

This is a **single self-contained HTML file** with no server, no build step, and no external dependencies beyond Google Fonts.

### Deploy in 30 seconds

```bash
# Clone the repository
git clone https://github.com/[your-org]/shem-hamephorash-wheel.git

# Open directly in any modern browser
open shem-hamephorash-wheel.html
```

Or drag the file into any browser window. That is all.

### Host on GitHub Pages

```bash
# From your repository root
git add shem-hamephorash-wheel.html
git commit -m "Deploy wheel"
git push origin main
# Enable GitHub Pages → root → main in repository Settings
```

The tool will be live at `https://[your-org].github.io/[repo-name]/shem-hamephorash-wheel.html`

### Requirements

- Any modern browser (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
- An internet connection on first load (for Google Fonts — Cinzel Decorative, IM Fell English, Uncial Antiqua, Frank Ruhl Libre)
- Works offline after fonts are cached

## Technical Stack

| Concern | Technology |
|---|---|
| Markup | HTML5 |
| Style | CSS3 with custom properties (CSS variables) |
| Logic | Vanilla JavaScript (ES2020) |
| SVG | Inline, procedurally generated by JS at runtime |
| Hebrew text | Frank Ruhl Libre (Google Fonts) + `direction: rtl; unicode-bidi: bidi-override` |
| Fonts | Cinzel Decorative · IM Fell English · Uncial Antiqua · Frank Ruhl Libre |
| Images | Three images embedded as base64 (Ana Bekoach, 231 Gates mandala, Cube of Space) |
| Dependencies | **None** (no frameworks, no npm, no bundler) |
| File size | ~720 KB (includes all images inline) |

The wheel SVG is generated at runtime by a `buildWheel()` function that calculates all arc paths, radial label positions, and rotation transforms from the data arrays. The 72-row data tables are stored as plain JavaScript arrays inside the script tag — no external JSON, no API calls.

## Customisation

Because the data is stored in plain JavaScript arrays at the top of the `<script>` tag, it is straightforward to:

- **Correct or extend angel attributes** — edit the `AA` (angel attribute) or `AF` (angel function) arrays
- **Add Tarot correspondences** — add a `TAROT` array indexed 0–71 and render it in `OG()`
- **Change the colour palette** — all colours are CSS custom properties in `:root {}`
- **Add a language** — swap the `Q83` array for any other 36-element text corpus
- **Export a gate** — extend `OG()` to generate a printable talisman card
